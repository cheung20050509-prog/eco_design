"""
【示例23】str.contains方法查询。
"""
"""
字符串中的 \w 是一个无效的转义序列。在 Python 中，反斜杠 \ 是转义字符，
因此 \w 会被解释为一个无效的转义序列，而不是正则表达式中的 \w（表示任意字母、数字或下划线）。
为了避免这个问题，我们可以通过在字符串前加上 r（表示原始字符串）来禁用转义字符的处理。
"""
import pandas as pd

# 创建一个数据框
df = pd.DataFrame({'name': ['华天科技', '海南高速', '汉王科技', '海南橡胶', '南国置业'], 
                   'price': [9.48, 5.89, 25.20, 4.61, 2.22]})

# 查询name列中包含“科技”的行
print('包含“科技”的行：\n', df[df['name'].str.contains('科技')])
print('-------------------------------')

# 查询name列中符合正则表达式'\w{1}南'的行：“南”前有一个字符的行
print('“南”前有一个字符的行：\n', df[df['name'].str.contains(r'\w{1}南', regex=True)])
print('-------------------------------')

# 查询name列中符合正则表达式'^南'的行：以“南”开头的行
print('以“南”开头的行：\n', df[df['name'].str.contains(r'^南', regex=True)])

"""
【示例29修改】绘制折线图：设置trade_date为X轴数据，close与open为Y轴数据。
原教材提供的代码中使用了 mpl.rcParams['figure.dpi']，但 mpl 并未被定义。
实际上，mpl 是 matplotlib 的常用别名，但在代码中并未导入或定义它。
修复这个问题，需要导入 matplotlib 并为其定义别名 mpl，
或者直接使用 plt.rcParams 来设置 figure.dpi。

第二个更严重的问题来自于数据文件第一列的日期格式：
日期格式中的月份和日期没有补充零。比如 2022/1/5 这种日期格式，pd.to_datetime() 默认可能无法直接识别。
因为pandas默认读取2022-01-05格式，如果用“stock_df['trade_date'] = pd.to_datetime(stock_df['trade_date'], format='%Y/%m/%d')”来
转换日期，那么月份和日期也要补充零，也就是“2022/01/05”才行，每一天的月份和日期必须是两位数。但是文件中的数据并没有补充零。
解决方案：一种灵活的解决方法是使用 errors='coerce' 来处理不符合预期格式的日期。如果日期格式不一致，
pd.to_datetime() 会将无法解析的部分转换为 NaT，并且会自动适应各种日期格式。
可以直接去掉 format 参数，让 pandas 自动推断日期格式。
"""

import pandas as pd

# 设置清晰度
